# API

#### 1. Сущности

**Трек (Track)**

| Поле       | Бизнес-описание                                                                      | Пример значения                        |
|------------|--------------------------------------------------------------------------------------|----------------------------------------|
| title      | Название трека                                                                       | "Вес"                                  |
| type       | Тип трека: `number`(числовой), `inc`(инкрементальный), `duration`(продолжительность) | "int"                                  |
| unit       | Единицы измерения(опциональное)                                                      | "кг"                                   |
| visibility | Видимость: `private`(личный), `internal`(групповой), `public`(виден всем),           | "internal"                             |
| owner      | Владелец трека                                                                       | "3fa85f64-5717-4562-b3fc-2c963f66afa6" |
| id         | Идентификатор трека                                                                  | 123456789                              |
| createDate | Дата начала трека в формате timestamp                                                | 1769964759                             |

**Данные трека (TrackRecord)**

| Поле    | Бизнес-описание                            | Пример значения |
|---------|--------------------------------------------|-----------------|
| trackId | Идентификатор трека                        | 123456789       |
| value   | Значение                                   | 84              |
| date    | Дата внесения значения в формате timestamp | 1769964759      |

#### 2. Бизнес-функции (методы)

**CRUDS для трека**

1. **`Track.create`** - Создание нового трека
    - *Бизнес-правило*: Все пользователи могут создавать новый трек. Единицы измерения не обязательны
      к заполнению. Видимость по умолчанию - `private`. Владелец по умолчанию - текущий пользователь. Идентификатор
      трека генерируется автоматически.
    - *Ограничения*: Минимум 3 символа в названии трека. Обязательно выбор типа трека.

2. **`Track.update`** - Обновление существующего трека
    - *Бизнес-правило*: Только владелец может изменять трек.
    - *Ограничения*: Возможно изменить только название, единицы измерения и видимость.

3. **`Track.read`** - Получение информации о треке
    - *Бизнес-правило*: Для `private`-трека доступ только для владельца. Для `internal`-трека доступ только определенной
      группе пользователей. Для `public`-трека доступ для всех пользователей.

4. **`Track.delete`** - Удаление трека
    - *Бизнес-правило*: Только владелец может удалить трек.

5. **`Track.search`** - Поиск трека
    - *Бизнес-правило*: Поиск трека возможен только среди `public`-треков.

**CRUDS для данных трека**

1. **`TrackRecord.add`** - Добавление данных в трек
    - *Бизнес-правило*: Только владелец может добавлять данные в трек.
    - *Ограничения*: Обязательно должно быть указано значение соответствующее `Track.type`.

2. **`TrackRecord.update`** - Обновление существующих данных трека
    - *Бизнес-правило*: Только владелец может изменять трек.

3. **`TrackRecord.read`** - Получение данных трека
    - *Бизнес-правило*: Данные `private`-трека доступы только для владельца. Данные `internal`-трека доступ только
      определенной группе пользователей. Данные `public`-трека доступ для всех пользователей.

4. **`TrackRecord.delete`** - Удаление данных трека
    - *Бизнес-правило*: Только владелец может удалить данные трека.

#### 3. Бизнес-правила

1. **Тип трека**:
   - `number`: В трек можно добавлять любые числовые значения
    - `inc`: Значение в треке изменяется на +1 или -1
    - `duration`: Трек учитывает продолжительность. В `TrackRecord.value` указывается время завершения.

2. **Видимость трека**:
    - `private`: Трек виден только владельцу
    - `internal`: Трек виден определенной группе пользователей
    - `public`: Видно всем пользователям
